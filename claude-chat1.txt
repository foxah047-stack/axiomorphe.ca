Just an XML file sitting on the server that lists your recent posts in a standardized format. Nothing dynamic, no database required. It looks like this:

<rss version="2.0">
  <channel>
    <title>CGBVRR — Nouvelles</title>
    <link>https://restigouche.org</link>
    <description>Mises à jour du bassin versant</description>

    <item>
      <title>Comptages de saumons 2025</title>
      <link>https://restigouche.org/nouvelles/saumons-2025</link>
      <pubDate>Fri, 08 May 2026 09:00:00 GMT</pubDate>
      <description>Résumé de l'article ici...</description>
    </item>

  </channel>
</rss>
You save that as feed.xml and link to it in the <head> of every page:
html<link rel="alternate" type="application/rss+xml" 
      title="CGBVRR Nouvelles" href="/feed.xml">
That's literally it. Feed readers detect it automatically from that tag.

The actual workflow for a small org
The honest answer is you don't want to hand-edit XML every time they post something. There are two practical approaches:
Option 1 — GitHub Pages + manual feed

Option 2 — Use a CMS that generates it automatically
If you build the site on something like Jekyll (which works natively with GitHub Pages) or Wordpress, RSS is generated automatically with zero extra work. Jekyll has a plugin called jekyll-feed — one line in your config file and it's done forever.
